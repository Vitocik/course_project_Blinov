# Nutrition Coach Mobile Application

## О проекте

**Nutrition Coach** — это кроссплатформенное мобильное приложение для подбора питания, тренировок и отслеживания прогресса пользователя.  
Проект разработан в рамках курсовой работы по дисциплине мобильной разработки.

Приложение позволяет:

- регистрироваться и авторизовываться через JWT;
- роли `USER` и `ADMIN`;
- создавать и редактировать планы питания;
- создавать и редактировать тренировки;
- просматривать рекомендации;
- отслеживать прогресс;
- поиск и CRUD-операции;
- работать в светлой и тёмной теме;
- использовать локальное кэширование для оффлайн-режима;
- взаимодействовать с REST API сервера.

# REST API

Серверная часть реализует REST API с документацией Swagger/OpenAPI.

## Основные эндпоинты

### Аутентификация

```http
POST /api/auth/register
POST /api/auth/login
```

### Meals

```http
GET    /api/meals
GET    /api/meals/{id}
POST   /api/meals
PUT    /api/meals/{id}
DELETE /api/meals/{id}
GET    /api/meals/search
```

### Workouts

```http
GET    /api/workouts
GET    /api/workouts/{id}
POST   /api/workouts
PUT    /api/workouts/{id}
DELETE /api/workouts/{id}
GET    /api/workouts/search
```

### Progress

```http
GET    /api/progress
POST   /api/progress
DELETE /api/progress/{id}
```

# JWT-аутентификация

После успешной авторизации сервер выдаёт JWT-токен.

Токен:

- используется для доступа к защищённым эндпоинтам;
- хранится локально через AsyncStorage;
- автоматически добавляется в Axios Interceptor.

---

# Локальное кэширование

Для оффлайн-режима используется AsyncStorage.

Кэшируются:

- пользовательские данные;
- список тренировок;
- список блюд;
- настройки темы;

# Установка и запуск проекта

# 1. Клонирование проекта

```bash
git clone https://github.com/USERNAME/course_project_Blinov.git
```

---

# 2. Запуск backend

## Переход в папку backend

```bash
cd backend
```

## Сборка проекта

```bash
mvn clean install
```

## Запуск сервера

```bash
mvn spring-boot:run
```

Сервер будет доступен:

```text
http://localhost:8080
```

---

# 3. Настройка PostgreSQL

Создать базу данных:

```sql
CREATE DATABASE nutrition_coach;
```

Указать настройки подключения в:

```text
backend/src/main/resources/application.yml
```

---

# 4. Запуск мобильного приложения

## Переход в папку mobile

```bash
cd mobile
```

## Установка зависимостей

```bash
npm install
```

## Настройка адреса backend для телефона

Для физического телефона удобнее всего использовать ngrok-туннель.  
Сначала запусти backend на компьютере, затем в отдельном окне:

```bash
ngrok http 8080
```

После запуска ngrok скопируй выданный `https://...ngrok-free.app` адрес и вставь его в поле `API URL` внутри приложения.

Если собираешь APK заранее, можно задать адрес через `mobile/.env` с переменной `EXPO_PUBLIC_API_URL`.

## Запуск Expo

```bash
npx expo start
```
