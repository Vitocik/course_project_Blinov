# CI/CD

## Что может входить
- сборка проекта;
- запуск тестов;
- проверка статического качества;
- публикация артефактов.

## Для курсовой
Достаточно описать логичную схему автоматической сборки backend и проверки тестов. В учебном проекте целесообразно использовать следующий сценарий:

1. При каждом обновлении в репозитории запускается сборка backend.
2. Выполняются unit- и integration-тесты.
3. Генерируется отчёт JaCoCo.
4. При успешной сборке артефакты могут быть использованы для демонстрации и защиты.

## Практический результат
Такой процесс повышает воспроизводимость проекта и делает его более похожим на реальное промышленное решение.
