# 03. Architecture

## Архитектурный подход
Проект построен по многослойной схеме:

- UI / Presentation Layer
- API Layer
- Service Layer
- Repository Layer
- Database Layer

## Плюсы архитектуры
- простая поддержка;
- разделение ответственности;
- удобное тестирование;
- понятный перенос логики между слоями;
- отсутствие жёсткой связки UI и backend.

## Технологии
### Mobile
- React Native
- Expo
- TypeScript
- Context API
- AsyncStorage
- Axios

### Backend
- Spring Boot
- Spring Security
- JPA / Hibernate
- PostgreSQL
- Swagger/OpenAPI
- JaCoCo

## Итог
Архитектура соответствует production-like MVP и подходит для защиты курсовой работы.
