# BUC (Business Use Case)

## Бизнес-акторы
| Актор | Роль |
|---|---|
| Пользователь | Ведёт личные данные о питании, тренировках и прогрессе |
| Администратор | Просматривает пользователей и служебную информацию |
| Система | Формирует планы и сохраняет данные |

## Основные бизнес-сценарии
1. **Регистрация и вход в систему**  
   Пользователь создаёт учётную запись и получает доступ к личным данным.

2. **Заполнение профиля**  
   Пользователь указывает параметры тела, цели и уровень активности.

3. **Получение плана питания**  
   Система на основе профиля и истории формирует рекомендации по питанию.

4. **Получение плана тренировок**  
   Система подбирает тренировочный план и сохраняет его в профиле пользователя.

5. **Ведение журнала прогресса**  
   Пользователь добавляет записи о весе и заметки, после чего видит динамику.

6. **Администрирование**  
   Администратор просматривает пользователей и служебные записи.

## Результат
BUC-уровень описывает ценность системы для пользователя: все ключевые действия сводятся к одному сценарию — планировать, сохранять и анализировать личный прогресс.
