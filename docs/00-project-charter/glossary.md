# Глоссарий

## Термины предметной области
- **Пользователь** — человек, который использует приложение для ведения питания и тренировок.
- **Администратор** — пользователь с расширенными правами доступа.
- **Профиль** — набор персональных параметров пользователя.
- **Цель** — желаемый результат, например снижение веса или поддержание формы.
- **Уровень активности** — характеристика регулярности физической нагрузки.
- **Блюдо** — элемент плана питания с калорийностью и составом.
- **Приём пищи** — запись о том, что и когда было съедено.
- **Тренировка** — набор упражнений или тренировочный план.
- **Упражнение** — отдельное действие внутри тренировки.
- **Прогресс** — запись о динамике веса, объёмов и самочувствия.
- **Рекомендация** — совет, сформированный системой на основе данных пользователя.
- **План питания** — структурированный набор блюд на период.
- **План тренировок** — структурированный набор тренировок на период.
- **История изменений** — набор прошлых записей, сохранённых в системе.
- **Оффлайн-режим** — сценарий, при котором часть данных доступна из локального кэша.
- **JWT** — токен, подтверждающий авторизацию пользователя.
- **REST API** — интерфейс обмена данными между mobile и backend.
- **DTO** — объект передачи данных между слоями приложения.
- **JPA-сущность** — класс, сопоставленный таблице базы данных.
- **Репозиторий** — слой доступа к данным.
- **Сервис** — слой, где реализована бизнес-логика.

## Примечание
Глоссарий собран отдельно, чтобы его было удобно использовать при защите и проверке документации.
