# Руководство администратора

## Роль ADMIN
Администратор имеет доступ к расширенным возможностям интерфейса и служебным данным.

## Доступные действия
- просмотр списка пользователей;
- контроль тестовых и демонстрационных данных;
- проверка работоспособности защищённых сценариев;
- демонстрация различий между ролями `USER` и `ADMIN`.

## Что удобно показать на защите
1. Вход под обычным пользователем.
2. Вход под администратором.
3. Доступность административного списка пользователей.
4. Различие доступных экранов и операций.

## Рекомендация
Для защиты удобно заранее подготовить два аккаунта: `USER` и `ADMIN`.

## Примечание по безопасности
Административные сценарии следует демонстрировать только в тестовой среде, а чувствительные данные — не публиковать в открытом виде.
