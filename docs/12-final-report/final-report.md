# Итоговый отчёт по курсовому проекту

![Пояснительная записка](../images/Курсовая_Блинов.docx)

## Тема проекта

Разработка мобильного приложения Nutrition Coach для контроля питания, физической активности и отслеживания прогресса пользователя

## Аннотация

В рамках курсового проекта было разработано мобильное приложение Nutrition Coach, предназначенное для помощи пользователям в формировании здорового образа жизни, контроле питания, планировании тренировок и отслеживании персонального прогресса.

Приложение предоставляет удобный интерфейс для ведения информации о рационе питания, физических упражнениях и достижениях пользователя. Архитектура решения предусматривает разделение клиентской и серверной частей, что обеспечивает масштабируемость и возможность дальнейшего развития системы.

## Цель проекта

Разработать мобильное приложение, позволяющее пользователям:

- контролировать рацион питания;
- вести учёт тренировок;
- отслеживать динамику изменения показателей;
- получать персонализированные рекомендации;
- хранить и анализировать историю активности.

## Задачи проекта

Для достижения поставленной цели были выполнены следующие задачи:

1. Проведен анализ предметной области.
2. Сформированы функциональные и нефункциональные требования.
3. Построены UML-диаграммы и архитектурные модели.
4. Разработана структура базы данных.
5. Реализован серверный API.
6. Разработан мобильный клиент.
7. Реализованы механизмы авторизации пользователей.
8. Выполнено тестирование основных сценариев использования.
9. Подготовлена проектная документация.

## Краткое описание реализации

### Мобильный клиент

Мобильное приложение реализовано на базе React Native и Expo.

Основные возможности:

- регистрация и авторизация пользователей;
- просмотр персонального профиля;
- управление планами питания;
- ведение журнала тренировок;
- отслеживание прогресса;
- получение рекомендаций.

### Серверная часть

Серверная часть обеспечивает:

- обработку запросов клиентов;
- выполнение бизнес-логики;
- хранение данных;
- управление пользователями;
- предоставление REST API.

## Архитектура решения

В проекте используется архитектурный подход PCMEF.

### Presentation

Отвечает за пользовательский интерфейс мобильного приложения.

### Control

Отвечает за обработку HTTP-запросов и взаимодействие клиента с сервером.

### Mediator

Содержит бизнес-логику приложения.

### Entity

Представляет бизнес-сущности системы:

- User
- Meal
- Workout
- NutritionPlan
- Progress

### Foundation

Обеспечивает доступ к данным.
