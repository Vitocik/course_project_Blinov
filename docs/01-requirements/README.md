# 02. Requirements

## Функциональные требования
- регистрация и авторизация;
- просмотр и редактирование профиля;
- просмотр персональных рекомендаций;
- CRUD для блюд;
- CRUD для тренировок;
- учёт прогресса;
- поиск и фильтрация;
- админ-панель;
- сохранение данных в offline cache;
- переключение темы оформления.

## Нефункциональные требования
- TypeScript на mobile;
- REST API на backend;
- JWT security;
- PostgreSQL;
- Swagger/OpenAPI;
- тестируемая архитектура;
- модульные тесты;
- производительность и стабильность;
- визуальная консистентность UI.

## Соответствие курсовой
Проект покрывает требования по:
- mobile stack;
- backend stack;
- минимум 5 экранов;
- CRUD;
- navigation;
- offline cache;
- loading/error/empty states;
- минимальному количеству эндпоинтов;
- тестовому покрытию.
