# Техническое задание

## 1. Основание для разработки
Курсовой проект по дисциплине мобильной разработки.

## 2. Наименование системы
Мобильное приложение **Nutrition Coach**.

## 3. Назначение разработки
Система предназначена для ведения питания, тренировок и отслеживания прогресса пользователя.

## 4. Требования к функциональности
### 4.1 Пользовательские функции
- регистрация и авторизация;
- просмотр и редактирование профиля;
- работа с блюдами;
- работа с тренировками;
- ведение прогресса;
- получение плана питания;
- получение плана тренировок;
- оффлайн-доступ к части данных.

### 4.2 Административные функции
- просмотр списка пользователей;
- доступ к расширенной информации по системе.

### 4.3 Требования к backend
- REST API;
- JWT-аутентификация;
- CRUD для ключевых сущностей;
- OpenAPI/Swagger;
- модульные тесты.

## 5. Требования к mobile
- поддержка экранов основных сценариев;
- хранение токена в локальном хранилище;
- поддержка светлой и тёмной темы;
- работа с локальным кэшем.

## 6. Требования к качеству
- разделение на слои;
- понятная структура проекта;
- наличие документации по этапам;
- наличие тестирования и отчётов качества;
- соответствие архитектурной модели PCMEF.

## 7. Критерии приёмки
Работа считается выполненной, если:
- приложение запускается;
- пользователь может войти в систему;
- доступны основные разделы;
- backend отвечает на запросы;
- документация оформлена по этапам;
- есть материалы для защиты;
- реализованы основные сценарии и зафиксированы в документации.
