# Domain Model

## Основные сущности
- `UserAccount`
- `UserProfile`
- `Meal`
- `WorkoutPlan`
- `ProgressEntry`

## Бизнес-логика домена
- пользователь проходит авторизацию и получает JWT;
- профиль хранит параметры тела и цели;
- блюда содержат КБЖУ и заметки;
- тренировки содержат длительность и расход калорий;
- прогресс хранит даты и массу тела;
- dashboard агрегирует данные профиля и прогресса.

## Логика персонализации
Система строит рекомендации на основе:
- возраста;
- пола;
- роста;
- веса;
- уровня активности;
- цели;
- количества тренировочных дней.
