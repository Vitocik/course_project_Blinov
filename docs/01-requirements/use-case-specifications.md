# Use Case Specifications

## 1. Регистрация
**Актор:** пользователь  
**Предусловие:** пользователь не авторизован  
**Основной поток:**
1. пользователь вводит email, пароль и имя;
2. приложение отправляет запрос на `/api/auth/register`;
3. backend создаёт аккаунт и профиль;
4. приложение сохраняет токен и данные пользователя.

## 2. Вход
**Актор:** пользователь  
**Основной поток:**
1. пользователь вводит email и пароль;
2. приложение отправляет запрос на `/api/auth/login`;
3. backend возвращает JWT;
4. mobile сохраняет данные в AsyncStorage.

## 3. Управление блюдами
**Актор:** пользователь  
**Основной поток:**
1. открыть список блюд;
2. создать/изменить/удалить запись;
3. данные синхронизируются с backend;
4. при отсутствии сети используется локальный кэш.

## 4. Управление тренировками
Аналогично блюдам, но с сущностью `WorkoutPlan`.

## 5. Отслеживание прогресса
Пользователь добавляет запись веса и заметку, после чего видит динамику в списке и на dashboard.
