# Test Plan

## Проверяемые области
- авторизация;
- профиль пользователя;
- dashboard;
- питание;
- тренировки;
- прогресс;
- роль администратора.

## Критерии успеха
- корректные ответы API;
- отсутствие критических ошибок;
- сохранение данных;
- предсказуемое поведение UI при ошибках и пустых состояниях.

## Дополнение

Раздел тестирования подтверждает, что проект проверен не только вручную, но и автоматическими тестами.  
Здесь особенно важно показать, какие сценарии покрыты unit- и integration-подходом, а также какие ветки логики уже закрыты JaCoCo.

- тест-план описывает, что именно нужно проверить;
- отчёт JaCoCo показывает объём покрытия;
- тесты на граничные случаи особенно полезны для branch coverage.

## Приоритет тестирования

1. авторизация и безопасность;
2. получение и изменение пользовательских данных;
3. работа с блюдами, тренировками и прогрессом;
4. административные сценарии;
5. обработка ошибок и пустых состояний.

## Ожидаемый результат

При выполнении тест-плана приложение должно корректно реагировать на валидные и невалидные запросы, а также сохранять данные без потери целостности.
