# ADR

## ADR-001: Использование PCMEF
Для backend выбрана многослойная архитектура PCMEF, поскольку она помогает разделить ответственность между контроллерами, бизнес-логикой, сущностями и доступом к данным. Архитектурное решение зафиксировано в проектной документации и используется как основа для дальнейшего сопровождения проекта.

## ADR-002: Использование REST и JSON
Для обмена между mobile и backend используется REST API с JSON, так как это упрощает интеграцию, тестирование и документирование. Решение выбрано для обеспечения совместимости между клиентом и сервером.

## ADR-003: Использование PostgreSQL
В качестве основной СУБД используется PostgreSQL как надёжное и хорошо поддерживаемое решение для учебного проекта. Для автоматических тестов и локальной разработки применяется H2.

## ADR-004: Использование JWT
Аутентификация реализована на JWT, чтобы исключить хранение сессий на сервере и упростить работу мобильного клиента. Решение поддерживает базовый сценарий регистрации и авторизации.

## ADR-005: Использование локального кэша
Часть данных сохраняется локально, чтобы сохранить доступ к информации при временной недоступности сети. Это повышает удобство использования приложения в условиях нестабильного соединения.

## ADR-006: Использование OpenAPI/Swagger
API документируется через OpenAPI/Swagger для упрощения тестирования и демонстрации взаимодействия между клиентом и сервером.
